# 用户使用说明

这是一套单人私有工作站。

它的目标不是把所有事情都塞进一个页面，而是把 4 类日常工作连起来：

- 规划：你接下来要做什么
- 知识：你在思考和沉淀什么
- 写作：你准备输出什么
- 归档：哪些内容值得长期保留和回看

如果你是第一次使用，先记住一件事：

- `Dashboard` 看全局
- `Knowledge` 记思考
- `Writing` 写内容
- `Planner` 推任务
- `Archive` 回看沉淀
- `Search / Command Center` 负责快速跳转和执行
- `Activity Hub` 负责按时间线和工作流回放

---

## 1. 第一次登录后先做什么

建议按这个顺序开始：

1. 打开 `Dashboard`
2. 看一遍当前的 `Today Focus`、`Quick Actions` 和 `Recent Activity`
3. 进入 `Settings`
4. 确认你的显示名称、主题、默认回放视角
5. 回到首页，从下面 3 个入口里选一个开始：
   - `新建笔记`
   - `新建草稿`
   - `新建任务`

如果你没有任何历史数据，最推荐从 `Knowledge` 开始。

原因很简单：

- 先记下正在想什么
- 再把其中一部分推进成草稿
- 最后把需要执行的事情挂成任务

## 2. 每个模块是做什么的

### Dashboard

这是你的总览页。

适合做的事：

- 看今天最值得推进的事情
- 看最近更新过的任务、笔记、草稿、归档
- 从快捷入口进入常用动作
- 找到已经跨模块连接起来的工作线程

适合什么时候用：

- 刚登录时
- 不知道下一步该做什么时
- 想重新进入某条工作线时

### Planner

这是你的规划区。

适合做的事：

- 新建任务
- 修改任务状态
- 看今天、本周、逾期任务
- 把任务和笔记、草稿关联起来

适合什么时候用：

- 你已经知道要做什么，准备开始执行时
- 你想把某条笔记或草稿转成可执行事项时

### Knowledge

这是你的知识仓库。

适合做的事：

- 记录思考
- 维护主题、领域、标签
- 回看最近更新的笔记
- 从笔记直接发起草稿或任务

适合什么时候用：

- 你正在想、正在整理、正在归纳时
- 你还不适合直接进入写作或执行阶段时

### Writing

这是你的写作区。

适合做的事：

- 新建草稿
- 编辑图文内容
- 插入图片和视频
- 发布文章
- 回看草稿和已发布内容

适合什么时候用：

- 你已经有成型表达想法时
- 你想把某条笔记推进成可发布内容时

### Archive

这是你的归档和回看区。

适合做的事：

- 看已经沉淀下来的文章和笔记记录
- 收藏重要内容
- 按时间线回看最近沉淀

适合什么时候用：

- 你想回看最近有哪些内容值得重开时
- 你想把“完成过的东西”重新拉回工作流时

### Search / Command Center

这是你的快速入口。

适合做的事：

- 全局搜索任务、笔记、草稿、文章、归档
- 从当前工作姿态快速发起动作
- 回到你最近固定的工作流

适合什么时候用：

- 你知道自己要找什么时
- 你不想一层层点导航时
- 你想用“命令台”方式进入工作时

### Activity Hub

这是你的回放中心。

适合做的事：

- 看最近活动
- 看跨模块线程
- 看历史时间线
- 按不同视角重进工作

适合什么时候用：

- 你中断了一段时间，想重新进入上下文时
- 你想按执行、思考、发布、历史来切换视角时

## 3. 推荐的首次使用路径

第一次使用，推荐按这条路径走：

### 路径 A：从笔记开始

1. 去 `Knowledge`
2. 新建一条笔记
3. 写下当前要整理的想法
4. 在笔记详情页点“从笔记开始草稿”
5. 进入 `Writing`
6. 把这条想法扩成草稿
7. 如果里面有需要执行的事情，再从笔记或草稿创建任务

这是最适合“我脑子里有东西，但还没整理成行动”的入口。

### 路径 B：从任务开始

1. 去 `Planner`
2. 新建任务
3. 如果任务背后有背景信息，就关联一条笔记或草稿
4. 在 `Planner` 里推进状态
5. 需要深挖时，跳回关联的 `Knowledge` 或 `Writing`

这是最适合“我已经知道要做什么”的入口。

### 路径 C：从写作开始

1. 去 `Writing`
2. 新建草稿
3. 写出结构、图片、视频内容
4. 发布文章
5. 发布后内容会进入 `Archive`
6. 如果文章背后还有待执行事项，再补一个任务

这是最适合“我已经准备开始表达”的入口。

## 4. 最常见的几种用法

### 用法 1：记一条想法，然后把它写出来

1. `Knowledge` 新建笔记
2. 在笔记页点“从笔记开始草稿”
3. `Writing` 中继续扩写
4. 发布
5. 到 `Archive` 回看

### 用法 2：从一条想法里拆出行动

1. `Knowledge` 打开一条笔记
2. 点“从笔记创建任务”
3. 在 `Planner` 里安排它
4. 后续执行时，任务还能跳回原笔记

### 用法 3：从草稿里拆出执行事项

1. `Writing` 打开草稿
2. 点“从草稿创建任务”
3. 在 `Planner` 中推进
4. 任务和草稿会保持关联

### 用法 4：快速回到正在做的工作

1. 打开 `Dashboard`
2. 看 `Recent Activity`
3. 看 `Cross-Module Threads`
4. 或直接去 `Activity Hub`

### 用法 5：直接搜索并跳转

1. 打开 `Search / Command Center`
2. 输入关键词
3. 直接进入对应任务、笔记、草稿、文章或归档记录

## 5. 如何理解 Activity Hub 和 Command Center

这是第一次使用时最容易觉得“有点复杂”的两个区域。

你可以先这样理解：

### Command Center

它是“快速操作台”。

当你知道自己要找什么、做什么时，用它：

- 搜索
- 快速跳转
- 从当前工作流发起动作

### Activity Hub

它是“工作回放台”。

当你不知道从哪重新接上上下文时，用它：

- 看最近发生了什么
- 看哪条工作线程仍然在推进
- 看有哪些东西已经进入归档

如果你只想先简单使用：

- 先把 `Command Center` 当成全局搜索
- 先把 `Activity Hub` 当成高级时间线

这样就够了。

---

## 6. 什么是“记忆工作流”

系统里有一套“remembered workflow / 记忆工作流”机制。

你可以把它理解成：

- 系统会记住你最近更偏向哪类工作
- 然后在首页、命令中心、活动中心里给你更顺手的入口

例如：

- 如果你最近一直在写作，系统会更偏向 `Writing`
- 如果你最近一直在整理笔记，系统会更偏向 `Knowledge`

这不是强制切换，只是帮助你更快回到当前节奏。

如果你不需要它：

- 可以在 `Settings` 或 `Modules` 里切换
- 也可以清空这个偏向，让系统回到更中性的状态

## 7. 如果你觉得复杂，应该怎么用

最简单的使用方式只有 3 条规则：

### 规则 1：先只用 3 个地方

先只用：

- `Dashboard`
- `Knowledge`
- `Writing`

先不要强迫自己立刻理解所有系统层入口。

### 规则 2：把 Planner 当成“执行列表”

如果你还不习惯复杂的工作流联动，就把 `Planner` 先当普通任务列表用。

等你自然需要的时候，再去关联笔记和草稿。

### 规则 3：把 Search 当成“全局查找”

刚开始不用把 `Command Center` 当成复杂控制台。

你完全可以先把它当成：

- 搜笔记
- 搜草稿
- 搜任务
- 直接跳转

这样就足够了。

---

## 8. 推荐的日常使用节奏

如果你准备长期用这套工作台，推荐这样安排：

### 开始工作前

1. 打开 `Dashboard`
2. 看 `Today Focus`
3. 看 `Recent Activity`
4. 决定今天主线是什么

### 工作进行中

1. 在 `Knowledge` 记录思考
2. 在 `Writing` 扩成内容
3. 在 `Planner` 推进行动

### 工作结束前

1. 回 `Activity Hub`
2. 看今天推进了哪些线程
3. 回 `Archive` 看有哪些内容已经沉淀

---

## 9. 一个最短的上手建议

如果你今天只想马上开始，不想看完整手册，就按下面做：

1. 去 `Knowledge` 新建一条笔记
2. 把你现在最想做的事写下来
3. 从这条笔记发起草稿
4. 如果需要执行，再从笔记或草稿发起任务
5. 回 `Dashboard` 看整体进度

这就是这套工作台最核心的使用方式。

## 10. 当前版本说明

当前项目状态是：

- 历史封版基线：`V6.1.0`
- 当前稳定发布线：`V9.0.0`
- 当前版本重点：Knowledge 编辑体验、Writing 封面选择、Settings 个人资料能力
- 单人私有工作站
- Web 桌面优先
- 核心工作流已经可用，并已上线 Linux 生产环境

目前已经稳定可用的核心能力包括：

- Knowledge 文档式编辑、Markdown 优先、Mermaid 预览、笔记记录与整理
- Writing 草稿、Markdown 块、封面图选择、发布、正式文章删除
- Planner / Todo 的新建、编辑、完成、重开、归档、删除
- Today / Upcoming / Done 任务视图
- 轻量筛选：All / High / Doing
- Search / Archive / Settings 基础工作流
- Settings 中头像、显示名、工作台短句可配置

当前最适合把它当成：

- 个人规划 + 知识 + 写作 + 归档 的统一台面

而不是：

- 团队协作平台
- 移动优先工具
- 超轻量单页笔记应用
