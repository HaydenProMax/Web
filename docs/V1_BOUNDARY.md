# V1 Boundary

## Completed In V1

### Product Shape

- single-user private workstation
- modular shell plus isolated modules
- desktop-first web experience
- replay-oriented navigation and re-entry

### Modules Ready In V1

- Planner
- Knowledge
- Writing
- Archive
- Activity Hub
- Dashboard
- Search / Command Center
- Settings
- Modules
- Auth

### End-to-End Loops Ready In V1

- note -> draft
- draft -> publish
- publish -> archive
- note -> task
- draft -> task
- search -> result -> source page
- dashboard/archive/activity replay re-entry

## Explicitly Deferred To Phase 2

### Multi-User Expansion

- user registration
- multiple accounts
- per-user password system
- invites and collaboration
- team/shared workspaces

### Planner Deepening

- calendar grid views
- recurring tasks
- reminders
- time blocking
- notification delivery

### Knowledge Deepening

- backlinks graph
- block references
- note version history
- advanced semantic retrieval

### Writing Deepening

- richer editor ergonomics
- uploaded video pipeline
- unpublish/delete flow
- publication settings and SEO controls

### System / Platform Deepening

- plugin installation flow
- external module packages
- import/export
- mobile-first optimization
- advanced analytics and trends

## Rule For New Work

If a change is required to keep the current first-deliverable loops stable, it belongs in V1.

If a change primarily adds depth, breadth, collaboration, or platform surface beyond those loops, it belongs in Phase 2.
